- Users table → Third (after all FKs to it are gone)
- Independent tables → Last

============================================================================
WHY FOREIGN KEY ERROR OCCURS
============================================================================

The error "#1217 - Cannot delete or update a parent row: a foreign key 
constraint fails" happens when:

1. phpMyAdmin executes SQL statements individually, not as a batch
2. SET FOREIGN_KEY_CHECKS = 0; doesn't apply to subsequent statements
3. Tables are dropped in wrong order (parents before children)

SOLUTION: Use the SQL in Option 1 above, which:
- Disables FK checks
- Drops all tables in ONE statement (comma-separated)
- Re-enables FK checks
- All executed as single transaction
